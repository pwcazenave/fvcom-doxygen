37  !------------------------------------------------------------------------------
38  ! Lagrangian particle tracking using linked lists
39  !
40  ! Current Capabilities (September 08, 2006)
41  ! 1.) Multiprocessor Formulation:
42  ! Subdomains will only traject particles contained in their interior
43  ! Particles passing across interprocessor boundaries are exchanged
44  ! between neighboring processors.
45  ! 2.) Restart Capability:
46  ! The code can be restarted and the particle output file will
47  ! act as if the code had run with intermission. Read
48  ! below for details.
49  ! 3.) General tracking: particles are trajected at each internal time step
50  ! (dti) using a 4 stage RK scheme and the (u,v,sigma) velocity field.
51  !
52  ! 4.) Output File Contents
53  ! particle position, (x,y,z, + sigma) as well as an associated scalar
54  ! variable at the particle position. Current options include salinity
55  ! temperature, densities, and eddy viscosities but can be modified
56  ! easily to provide user defined fields such as a biological quantitiy
57  ! or tracer concentration.
58  !
59  ! 5.) initial particle position file. This file is in netcdf format.
60  ! It must contain information about number of particles, location
61  ! (x,y,z) as well as global parameters including an information
62  ! string, time of last output, number of outputs made. The last
63  ! two parameters are used when restarting the model. To create
64  ! an initial position file, start from the create_initial_lagpos.f90
65  ! utility supplied in the utilities folder of your FVCOM distribution.
66  ! You can use this script to easily convert an ascii particle file
67  ! to the FVCOM netcdf format or you can generate a grid of particles
68  ! within a user defined box.
69  !
70  ! Future Capabilities
71  ! 1.) Horizontal Random Walk.
72  ! To mimic the affects of turbulent eddies, we have use these algorithms.
73  ! They are quite varied, a bit hacky, and have the dangerous property of
74  ! perhaps violating local CFL conditions which will mean that particles
75  ! passing through domain boundaries will be forever lost. A particle
76  ! must first be detected inside a domain's halo element to know that it should
77  ! be sent to the neighboring processor who contains that particular element
78  ! in their interior. If it jumps right into the other processors interior.
79  ! It will be lost. There are ways of dealing with this, of course, but I don't
80  ! suspect it will happen anytime soon.
81  !
82  ! 2.) Vertical random walk.
83  ! This will be easier to implement, but the wide variety of models (read:
84  ! a lot of parameters) will mean a separate inputfile may be necessary
85  ! for their specification. Probably coming soon.
86  !
87  ! =======Notes====
88  ! Note: there is no specific array holding the particles because
89  ! they may traverse from one processor subdomain to another.
90  ! The particles are stored in the linked list 'particle_list'.
91  ! Particles may be added or deleted from the list (see linklist.F)
92  ! To access a particle property, for example, id, loop through the
93  ! linked list as follows:
94  !
95  ! 1.) lp => particle_list%first%next
96  ! 2.) do
97  ! 3.) if(.not. associated(lp) ) exit
98  ! 4.) write(*,*)'this is the current particles id: ',lp%v%id
99  ! 5.) lp => lp%next
100  ! 6.) end do
101  !
102  ! A list is comprised of 0 or more nodes
103  ! Line 1 sets the pointer to the first node in the list. (lp)
104  ! Line 2 starts an infinite loop
105  ! Line 3 exits when the end of the list is reached
106  ! Line 4 shows example usage: note lp is the node and lp%v is the particle
107  ! Line 5 move to the next node, without this line you have an infinite loop
108  ! Line 6 is the end of the loop
109  !
110  ! Reference: Object Oriented Programming via Fortran 90/95 , Ed Akin
111  !
112  ! Note, you must include NETCDF_IO in the Makefile to build the Lagrangian library
113  ! This will require you to build the netcdf librarie for your system.
114  ! This is a result of a code modernization effort and is part of conspiracy
115  ! against 'old school' programmers. Well, maybe it is a conspiracy, but
116  ! tough luck.
117  !
118  ! Runtime Parameters (to be placed in file: casename_run.dat):
119  ! Must be capital letters.
120  !
121  ! LAG_INTERVAL: output interval in model hours
122  ! LAG_INPFILE: input file name (will read INPDIR directory)
123  ! LAG_RESFILE: restart file (will dump to executable directory)
124  ! LAG_OUTFILE: output file (will write to OUTDIR/netcdf directory)
125  ! LAG_SCAL: scalar variable to include with output
126  ! LAG_COLD_START logical (T/F) giving startup type
127  !
128  ! LAG_COLD_START = T
129  ! --> a new outputfile will be created
130  ! --> initial positions will be output to outputfile
131  ! --> dumps are made every subsequent LAG_INTERVAL to outputfile
132  !
133  ! LAG_COLD_START = F (assumes input file generated by restart dump)
134  ! --> new outputfile will not be created
135  ! --> time of last dump is obtained from input file
136  ! --> stack # of last dump obtained from input file
137  ! --> next dump made to stack# + 1 at time of last dump + LAG_INTERVAL
138  ! --> initial positions not dumped to file (the assumed initial position
139  ! would be the first output in the file.
140  !
141  ! LAG_SCAL
142  ! current choices
143  ! s1 (salinity at nodes)
144  ! t1 (temperature at nodes)
145  ! rho1 (density at nodes)
146  ! km (eddy viscosity)
147  ! kh (vertical eddy diffusivity)
148  ! user_defined
149  ! --> users must set variable names in get_ud_scalnames
150  ! --> users must supply 3D field in add_scalar
151  ! --> recompile after modification!
152  ! search the code for "users" to see where to modify
153  !
154  ! General information on restarting the code.
155  ! If you restart the code but don't want to interrupt your particle output
156  ! file:
157  !
158  ! 1.) run the code. It will dump a particle restart file (designated by
159  ! LAG_RESFILE in the casename_run.dat file), every IRESTART iterations.
160  ! It will also dump one at the end of the run.
161  ! 2.) move the LAG_RESFILE to your INPDIR directory. Rename if you want.
162  ! Let's say you name it brainless_coding.nc
163  ! 3.) modify the following parameters in your runtime file
164  ! RESTART = hot_start (may already be set to this)
165  ! LAG_COLD_START = F
166  ! LAG_INPFILE = brainless_coding.nc
167  ! 4.) restart the code.
168  !
169  ! The restart file you used as your LAG_INPFILE contains information about
170  ! when the last output was made and the stack# of the output so that the
171  ! Lagrangian tracking module knows when to make the last dump and what
172  ! stack# to use in the netcdf file. By setting LAG_COLD_START=F, the code
173  ! knows not to create a new outputfile, but rather use the othe one.
174  !
175  ! If you do something like restart the code but restart with a different
176  ! number of particles but use LAG_COLD_START = F, I cannot be responsible
177  ! for what might happen. In simulations performed during the extensive
178  ! testing procedure used for every line of FVCOM code, we found that results
179  ! can range from crashed code to broken keyboards.
180  !

827  if(lp%v%zloc < -lp%v%h) lp%v%zloc = -lp%v%h

938  !==============================================================================|
939  subroutine traject(deltat,now,u1,u2,v1,v2,w1,w2,hl,el)
940  !==============================================================================|
941  ! integrate particle position from x0 to xn using velocity fields at time |
942  ! t0 and time tn |
943  ! deltat: time step between calls to traject (usually dti) |
944  ! time: model time in hours
945  ! u1/v1/w1: velocity field (u,v,omega) at time t0 |
946  ! u2/v2/w2: velocity field (u,v,omega) at time tn |
947  ! hl: bathymetry |
948  ! el: free surface elevation at time tn |
949  !==============================================================================|
950  use mod_prec
951  use lims
952  use control, only : iint
953  implicit none
954  real(sp), intent(in) :: deltat
955  TYPE(time), intent(in) :: now
956  real(sp), allocatable, intent(in) :: u1(:,:),u2(:,:),v1(:,:),v2(:,:)
957  real(sp), allocatable, intent(in) :: w1(:,:),w2(:,:)
958  real(sp), allocatable, intent(in) :: hl(:),el(:)
959  !------------------------------------------------------------------------------|
960  ! n : stage counter |
961  ! ns : number of stages in explicit runga-kutta |
962  ! chix : stage function evaluation for x-velocity |
963  ! pdx : stage particle x-position |
964  ! ul : stage u velocity |
965  ! eps : parameter defining depth of dry element |
966  ! dmax : maximum sigma depth |
967  ! a_rk : erk coefficients (a) |
968  ! b_rk : erk coefficients (b) |
969  ! c_rk : erk_coefficients (c) |
970  !------------------------------------------------------------------------------|
971  integer :: ns, status,ierr
972  integer, parameter :: mstage = 4
973  real(sp), parameter :: eps = 1.0e-5
974  real(sp), parameter :: dmax = -1.0_dp
975  real(sp), dimension(3) :: xtmp,xm
976  real(sp), parameter, dimension(mstage) :: a_rk = (/0.0_dp,0.5_dp,0.5_dp,1.0_dp/)
977  real(sp), parameter, dimension(mstage) :: b_rk = (/1.0_dp/6.0_dp,1.0_dp/3.0_dp, &
978  1.0_dp/3.0_dp,1.0_dp/6.0_dp/)
979  real(sp), parameter, dimension(mstage) :: c_rk = (/0.0_dp,0.5_dp,0.5_dp,1.0_dp/)
980  type(link_node), pointer :: p
981 
982  ! Model velocity at RK stage
983  real(sp), allocatable :: ul(:,:),vl(:,:),wl (:,:)
984  !------------------------------------------------------------------------------|
985 
986  !------------------------------------------------------------
987  ! allocate space for RK stage velocity
988  !------------------------------------------------------------
989  ALLOCATE(ul(0:nt,kb),vl(0:nt,kb),wl(0:mt,kb),stat=status)
990  IF(status/=0) CALL fatal_error("TRAJECT: COULD NOT ALLOCATE SPACE")
991 
992 
993  !--set particle time step (0 if particle is not released or exceeded T_end)
994  p => particle_list%first%next
995  do
996  if(.not. associated(p) ) exit !end of list, exit
997  if(now >= p%v%tbeg .and. now <= p%v%tend)then
998  p%v%deltat = deltat
999  else
1000  p%v%deltat = 0.0_sp
1001  endif
1002  p => p%next !set object
1003  end do
1004 
1005 
1006 
1007  !--initialize stage functional evaluations
1008  p => particle_list%first%next
1009  do
1010  if(.not. associated(p) ) exit !end of list, exit
1011  p%v%chi = 0.0_sp
1012  p => p%next !set object
1013  end do
1014 
1015 
1016  !--loop over Runge-Kutta stages and calculate stage velocities
1017  do ns=1,mstage
1018 
1019  !!particle position at stage n (x,y,sigma)
1020  IF(ns > 1)THEN
1021  p => particle_list%first%next
1022  do
1023  if(.not. associated(p) ) exit !end of list, exit
1024 !---------------------------------------------------------------
1025 # if defined(SPHERICAL)
1026 !---------------------------------------------------------------
1027  xm = a_rk(ns)*p%v%deltat*p%v%chi(:,ns-1)
1028  p%v%x(1) = p%v%xn(1) + xm(1)/(tpi*cos(p%v%xn(2)) + 1.0e-6)
1029  p%v%x(2) = p%v%xn(2) + xm(2)/tpi
1030  p%v%x(3) = p%v%xn(3) + xm(3)
1031 
1032  IF(p%v%x(1)<0.0_sp) THEN
1033  p%v%x(1) = p%v%x(1) +360.0_sp
1034  ELSEIF(p%v%x(1)>360.0_sp) THEN
1035  p%v%x(1) = p%v%x(1) -360.0_sp
1036  END IF
1037 
1038  IF(p%v%x(2) > 90.0_sp) THEN
1039  p%v%x(2) = 180.0_sp - p%v%x(2)
1040  ELSEIF(p%v%x(2) < -90.0_sp) THEN
1041  p%v%x(2) = -180.0_sp - p%v%x(2)
1042  END IF
1043 
1044 
1045 # else
1046 
1047  p%v%x = p%v%xn + a_rk(ns)*p%v%deltat*p%v%chi(:,ns-1)
1048 !---------------------------------------------------------------
1049 # endif
1050 !---------------------------------------------------------------
1051  p => p%next !set object
1052  end do
1053  END IF
1054 
1055 
1056  !!determine element location
1057  p => particle_list%first%next
1058  do
1059  if(.not. associated(p) ) exit !end of list, exit
1060  p%v%elem = find_element_containing(p%v%x(1),p%v%x(2),p%v%elem)
1061  if (p%v%elem == 0) p%v%found = .false.
1062  p => p%next
1063  end do
1064 
1065  CALL delete_not_found(particle_list)
1066 
1067  !!shift particles to new domain if they have crossed boundary
1068  IF(par) call exchange_particles
1069 
1070 
1071  !!determine el/h at stage position to convert to sigma
1072  p => particle_list%first%next
1073  do
1074  if(.not. associated(p) ) exit
1075  if(p%v%elem /= 0) then
1076  p%v%el = interp_anodal(p%v%x(1),p%v%x(2),p%v%elem,el)
1077  p%v%h = interp_anodal(p%v%x(1),p%v%x(2),p%v%elem,hl)
1078  end if
1079  p => p%next
1080  end do
1081 
1082  !!adjust sigma position of particle at boundaries
1083  p => particle_list%first%next
1084  do
1085  if(.not. associated(p) ) exit !end of list, exit
1086  p%v%x(3) = max(p%v%x(3),-(2.0_sp+p%v%x(3))) !mirror bottom
1087  p%v%x(3) = min(p%v%x(3),0.0_sp) !don't penetrate free surf
1088  p => p%next !set object
1089  end do
1090 
1091  !!calculate velocity field for stage n using c_rk coefficients
1092  ul = (1.0_sp-c_rk(ns))*u1 + c_rk(ns)*u2
1093  vl = (1.0_sp-c_rk(ns))*v1 + c_rk(ns)*v2
1094  wl = (1.0_sp-c_rk(ns))*w1 + c_rk(ns)*w2
1095 
1096  p => particle_list%first%next
1097  do
1098  if(.not. associated(p) ) exit
1099  p%v%u = interp_azonal(p%v%x(1),p%v%x(2),p%v%x(3),kbm1,p%v%elem,ul)
1100  p%v%v = interp_azonal(p%v%x(1),p%v%x(2),p%v%x(3),kbm1,p%v%elem,vl)
1101  p%v%w = interp_anodal(p%v%x(1),p%v%x(2),p%v%x(3),kb,p%v%elem,wl)
1102  p => p%next
1103  end do
1104 
1105  ! THE POSITION, p%v%x has not changed since we last did this?
1106  ! Do we need to update EL and H?
1107 
1108  !!calculate u,v,w at stage ns
1109  p => particle_list%first%next
1110  do
1111  if(.not. associated(p) ) exit !end of list, exit
1112  p%v%chi(1,ns) = p%v%u
1113  p%v%chi(2,ns) = p%v%v
1114  p%v%chi(3,ns) = p%v%w/(p%v%h+p%v%el) !delta_sigma/deltat = omega/d
1115  p => p%next !set object
1116  end do
1117 
1118  !!do not allow vertical motion in very shallow water
1119  p => particle_list%first%next
1120  do
1121  ! THIS SHOULD NEVER HAPPEN, eps IS TOO SMALL?
1122  if(.not. associated(p) ) exit !end of list, exit
1123  if((p%v%h + p%v%el) < eps) p%v%chi(3,:) = 0.0_sp
1124  p => p%next !set object
1125  end do
1126 
1127  end do !stage loop
1128 
1129  !-sum stage contributions to get updated particle positions-------------------!
1130  !-store accumulated pathlength
1131  p => particle_list%first%next
1132  do
1133  if(.not. associated(p) ) exit !end of list, exit
1134 
1135 !---------------------------------------------------------------
1136 # if defined(SPHERICAL)
1137 !---------------------------------------------------------------
1138  xtmp = 0.0_sp
1139  do ns=1,mstage
1140  xm = p%v%deltat*p%v%chi(:,ns)*b_rk(ns)
1141 
1142  xtmp = xtmp + xm
1143 
1144  p%v%xn(1) = p%v%xn(1) + xm(1)/(tpi*cos(p%v%xn(2)) + 1.0e-6)
1145  p%v%xn(2) = p%v%xn(2) + xm(2)/tpi
1146  p%v%xn(3) = p%v%xn(3) + xm(3)
1147 
1148 
1149 
1150  IF(p%v%xn(1)<0.0_sp) THEN
1151  p%v%xn(1) = p%v%xn(1) +360.0_sp
1152  ELSEIF(p%v%xn(1)>360.0_sp) THEN
1153  p%v%xn(1) = p%v%xn(1) -360.0_sp
1154  END IF
1155 
1156  IF(p%v%xn(2) > 90.0_sp) THEN
1157  p%v%xn(2) = 180.0_sp - p%v%xn(2)
1158  ELSEIF(p%v%xn(2) < -90.0_sp) THEN
1159  p%v%xn(2) = -180.0_sp - p%v%xn(2)
1160  END IF
1161 
1162 
1163 
1164  end do
1165  p%v%pathlength = p%v%pathlength + sqrt(xtmp(1)**2 + xtmp(2)**2)
1166 
1167 # else
1168  xtmp = p%v%xn
1169  do ns=1,mstage
1170  p%v%xn = p%v%xn + p%v%deltat*p%v%chi(:,ns)*b_rk(ns)
1171  end do
1172  p%v%pathlength = p%v%pathlength + sqrt( (p%v%xn(1)-xtmp(1))**2 + (p%v%xn(2)-xtmp(2))**2)
1173 !---------------------------------------------------------------
1174 # endif
1175 !---------------------------------------------------------------
1176 
1177  p => p%next !set object
1178  end do
1179 
1180  !-adjust sigma locs near top/bottom-------------------------------------------!
1181  p => particle_list%first%next
1182  do
1183  if(.not. associated(p) ) exit !end of list, exit
1184  p%v%xn(3) = max(p%v%xn(3),-(2.0+p%v%xn(3) )) !mirror bottom
1185  p%v%xn(3) = min(p%v%xn(3),0.0_sp) !don't penetrate free surf
1186  p => p%next !set object
1187  end do
1188 
1189  !-update x location and find the element -------------------------------------!
1190  p => particle_list%first%next
1191  do
1192  if(.not. associated(p) ) exit !end of list, exit
1193  p%v%x = p%v%xn
1194  p%v%elem = find_element_containing(p%v%x(1),p%v%x(2),p%v%elem)
1195  if (p%v%elem == 0) p%v%found = .false.
1196  p => p%next
1197  end do
1198 
1199  CALL delete_not_found(particle_list)
1200 
1201  !--evaluate bathymetry and free surface height at updated particle position----!
1202  p => particle_list%first%next
1203  do
1204  if(.not. associated(p) ) exit
1205  p%v%el = interp_anodal(p%v%x(1),p%v%x(2),p%v%elem,el)
1206  p%v%h = interp_anodal(p%v%x(1),p%v%x(2),p%v%elem,hl)
1207  p%v%zloc = (p%v%x(3)*(p%v%h+p%v%el) + p%v%el) !particle depth (m)
1208  p => p%next
1209  end do
1210 
1211  DEALLOCATE(ul,vl,wl)
